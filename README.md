# tw-block-warrant — 鉅額交易方向判定 × 權證同向訊號

台股（上市）**鉅額交易**買賣方向推定 × **權證資金流**同向訊號的獨立策略專案。
每交易日 **19:00（台灣時間）** 自動抓取當日資料、累積歷史、判定方向、更新報表。

📊 **每日報表**：<https://appr1ciat1.github.io/tw-block-warrant/>（= `report.html`）

## 兩層架構（獨立分開，互不取代）

專案分為兩個**獨立版本**，延伸層只唯讀原始層的資料、絕不改動或取代原始版：

| 層 | 位置 | 內容 | 執行 |
|---|---|---|---|
| **① 原始每日訊號版** | repo 根目錄 | 抓資料、判方向、五窗輪動訊號 → `report.html`/`index.html`/`signals.json`。這是每天 19:00 自動跑、Pages 首頁呈現的**生產系統**，邏輯保持穩定。 | `update.py`（workflow 自動） |
| **② 延伸研究/優化版** | `research/` | 回測引擎、設定掃描、風報比優化等**研究層**，唯讀根目錄資料做分析，輸出 `research/backtest.html`。不進每日 workflow、不動原始版訊號。 | 手動 `python research/*.py` |

原始版的每日訊號定義（判定矩陣）保持不變；優化得到的參數（如共振≥3、持有20日）
屬**延伸版的策略配置**，是原始版之上另建的一層，不回寫覆蓋原始版。

## 核心假說

一檔股票的認購權證被**連續買進且量大**，是槓桿多頭資金流入的訊號。
若同期該股的**鉅額交易方向也是買進**（大戶現貨吸籌），現貨與槓桿資金**同向** → 跟進買入。
反之，鉅額方向是**賣出**但權證卻有買盤，是典型「**賣現貨、買權證**」避險換倉——
權證買盤非新多頭資金，一律剔除。方向判定讓我們**只剔除真正的避險形態**，
而不是把所有有鉅額成交的股票一律排除。

## 關鍵問題：鉅額交易不揭露買賣方，方向怎麼判定最正確？

TWSE 只公布鉅額交易的成交價/量，不公布買賣雙方身分。可用的推定證據依可靠度排序：

| 層級 | 證據 | 資料 | 判讀 | 本專案 |
|---|---|---|---|---|
| 1 | **同日三大法人買賣超**（T86，統計**含**鉅額交易） | T86 | 當日法人買超金額 ≈ 鉅額金額 → 法人大概率是**買方**（吸籌）；賣超相當 → 法人是**賣方**（出貨） | ✅ 自動 |
| 2 | **溢折價**（成交價 vs 當日收盤價） | BFIAUU + MI_INDEX 收盤價 | 買方急著要貨→**溢價**成交；賣方急著出貨→**折價**成交；**貼平盤**（尤其配對交易）多為關係人移轉/節稅/ETF 實物申贖→**中性** | ✅ 自動 |
| 3 | **董監事持股轉讓申報**（MOPS） | 公開資訊觀測站 | 事前申報「以鉅額交易方式轉讓」→ **賣方=內部人**，方向確定（覆蓋率低但最準） | 人工覆核 |
| 4 | **事後價格行為** | 收盤價歷史 | 買方主導的鉅額有正向永久價格衝擊 | `validate.py` 事後驗證 |
| — | 券商分點買賣明細 | TWSE 不公開 | 最直接但需付費第三方 | ✗ |

法人比對之所以有效，是因為 **T86 的法人買賣超統計「含」鉅額交易**：
外資若透過鉅額接下 5 億元持股，當日外資買超就會出現同量級的數字。

**逐筆方向分數** = (溢折價證據 ±1/0 + 法人證據 ±1/0) ÷ 2 ∈ [-1, +1]
**每檔標的** = 視窗內逐筆以成交金額加權平均 → `buy`（≥+0.25）/ `sell`（≤−0.25）/ `neutral`。
兩證據矛盾或貼平盤移轉 → 中性，不做方向宣稱（保守）。

> **首次實證（2026-07-03，60 交易日 / 2051 筆，樣本淺僅供參考）**：
> 法人比對方向單調有效（買組全期間勝過賣組）；溢折價「單獨用」在台股反向
> （大單折價常是買方接量要求的讓利，非賣方急出）；**「雙證據同向買」是唯一
> 10 日中位數為正（+0.29%）且勝率 >50% 的組**——即 🟢 訊號的組合。
> 資料每日累積後定期重跑 `validate.py` 覆核。

## 資料源（TWSE，僅上市）與儲存架構

| 資料 | 端點 | 累積目錄 |
|---|---|---|
| 鉅額交易日成交資訊（單一證券） | `rwd/zh/block/BFIAUU` | `data/block_trades/` |
| 全市場權證行情按標的聚合（認購 0999 / 認售 0999P） | `rwd/zh/afterTrading/MI_INDEX` | `data/warrant_flows/` |
| 全市場個股收盤價（溢折價基準＋事後驗證） | `MI_INDEX type=ALLBUT0999` | `data/stock_closes/` |
| 三大法人買賣超（統計含鉅額） | `rwd/zh/fund/T86` | `data/inst_flows/` |

MI_INDEX 權證表每列自帶「標的代號」，直接聚合到標的股，不需另抓權證基本資料。
每日判定結果（含被剔除組）append 至 `data/signal_history/`，永久累積。

**儲存 = 年度分檔（`data/<名稱>/YYYY.csv`）＋ 滾動留存 10 年（`--keep-years`）**：
單一 CSV 累積 10 年會超過 GitHub 單檔 100MB 硬限制（權證流 ~11MB/年），
且每日 commit 重寫大檔會讓 repo 膨脹；分檔後舊年檔永不變動、留存政策只刪整年檔。
每日訊號計算只切近 200 天資料（深歷史留給 `validate.py`），loading 恆定不隨年數變重。

**自動補洞（`--heal-days 45`）**：TWSE 限流時會回假錯誤（stat≠OK），會被當成
非交易日靜默跳過而留洞（實例：T86 2026-06-04 短列）。每日執行時以四資料源近 45 天
日期聯集為參考，自動補抓缺日。

**資料品質記事**：BFIAUU JSON 每日末列是「總計」彙總列，2026-07-03 前的抓取
曾誤存為交易（金額整整多一倍），已修正並全史清除；fetch 端已過濾。

## 時間段輪動偵查窗

每日對**五個偵查窗**各判定一次，判定點全部累積（`signal_history.csv` 的 `window` 欄）：

| 窗 | 定義 | 用意 |
|---|---|---|
| 近 5 交易日 | 最近一週 | 最即時的形態 |
| 近 10 交易日（主窗） | 最近兩週 | 報表主表 |
| 近 15 交易日 | 最近三週 | 中期累積 |
| 近 20 交易日 | 最近一個月 | 慢燉吸籌/出貨 |
| W3（週三→下週三） | 上個週三起的當輪 | 錨定週選擇權結算週期 |

報表的「⏱ 多窗共振」矩陣顯示每檔在五窗的判定：**🟢 共振 ≥ 2 窗 = 同向買入跨窗成立
（更可信）**；🚫 共振 ≥ 2 窗 = 避險換倉形態跨窗成立。累積足夠後可用
`signal_history.csv` 統計「哪個窗長最有預測力」。（`--windows`／`--primary-window` 可調）

| API | 可回溯至 | 備註 |
|---|---|---|
| BFIAUU 鉅額 | **2005-04-04**（API 明示下限） | |
| T86 三大法人 | **2012-05-02** | 欄位三代變革 12→16→19 欄，parser 已相容 |
| MI_INDEX 收盤價 | ≤ 2005-01-04（實測 OK） | |

## 判定矩陣（母體 = 偵查窗內有鉅額成交的標的）

權證腿：A 連買 = 認購「價漲成交金額 > 價跌成交金額」連續 ≥ 3 日；
B 量多 = 當日認購金額 ≥ 2.0× 前 20 日中位數；（且視窗認購額 ≥ 10M 流動性底線）

| 鉅額方向 \ 權證 | 強（A+B） | 偏買 | 無買盤/看空 |
|---|---|---|---|
| **買** | 🟢 **同向買入** | 🟡 偏買 | ⚪ 不明 |
| **中性**（移轉類） | 🟡 偏買 | ⚪ 不明 | ⚪ 不明 |
| **賣** | 🚫 **避險換倉剔除** | 🚫 避險換倉剔除 | 🔴 出貨避開 |

分數 = 連買天數 × min(量能倍數, 5) × (1 + 鉅額方向分數)，僅供排序。

## 使用

```bash
pip install -r requirements.txt
python update.py                  # 每日增量更新 + 出報表（五窗輪動）
python update.py --backfill 90    # 首次回補 90 個日曆日
python update.py --no-update      # 只用既有 CSV 重出報表
python update.py --windows 5,10,20 --primary-window 5   # 自訂偵查窗
python update.py --deepen-to 2025-07-01   # 歷史往過去加深一年
python update.py --prem-th 0.01 --inst-ratio 0.3   # 調方向門檻
python validate.py                # 事後驗證：各方向組的後續超額報酬
python research/backtest.py --min-reso 3 --hold 20   # 事件驅動回測（權益曲線+Sharpe/MDD/Calmar）
python research/sweep.py          # 一次回放掃多組設定，比較風報比堆疊
```

輸出：`report.html` / `index.html`（同向買入 / 避險剔除 / 全部判定 / 逐筆明細含方向證據）
+ `signals.json`（`buy_signals`、`excluded_hedge`、`block_directions` 供程式串接）。

## 回測引擎（`backtest.py` / `sweep.py`）

把每日訊號變成真實組合的權益曲線與風報比指標——所有優化的量化基礎。
標準重疊組合法：訊號用 D 日資料、D+1 收盤成交（無前視），持有 N 日，含台股來回成本
0.585%，對標 0050。除權息以 ±10.5% 中性化處理（無還原股價下的近似，未計入配息 →
對高殖利率標的略偏保守；接還原股價為後續精化）。

**一年(2025-07→2026-07)實證結論**（單一多頭 regime，深挖後需重驗）：
- **持有期是最大槓桿**：持有 5 日 Sharpe −0.01（訊號未發酵）、10 日 2.10、**20 日 2.75**。
  訊號半衰期 5–10 日、20 日達峰 → 這是**慢訊號，抱 ~20 日**。
- **共振≥3 提升風報比**：Calmar 4.56→5.58（持有10）。
- **最佳組態 🟢·共振≥3·持有20**：年化 +117% / Sharpe **2.80** / MDD −14% / Calmar 8.61。
- 同期 0050 = Sharpe 3.32 / MDD −11%（AI 帶動的罕見平滑狂牛）。**策略絕對風報比很強
  但未在此多頭勝過大盤**——訊號的相對選股優勢（避開 ⚪ 組後續 −5%）需在空頭/震盪
  regime 才顯現，故深挖 2022 熊市後的重驗是關鍵。

## 自動化

`.github/workflows/update.yml`：每週一~六 UTC 11:00（**台灣 19:00**）執行，
XTAI 行事曆判定非交易日自動跳過；抓四份資料 → 判定 → 更新報表 → commit。
（GitHub cron 可能延遲數分鐘，實際落地約 19:00–19:15。）

## 限制

- 僅上市（TWSE）；上櫃（TPEx）權證與鉅額交易未納入。
- 鉅額方向為溢折價＋法人流**推估**，非事實揭露；盤中成交的鉅額以收盤價
  當基準有誤差（盤後 13:30–17:00 議價的鉅額則基準精確）。
- 法人比對是「同日流量」歸因：大型股當日法人巨量進出可能與該筆鉅額無關。
- 權證漲跌 × 成交金額是買賣方向 proxy，非逐筆內外盤資料。
- signal_history 於 2026-07-03 起使用現行分類（same_dir_buy / lean_buy /
  hedge_suspect / sell_avoid / unclear），之前的舊分類（buy_strong…）保留原樣。
